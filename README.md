# IAPY — Local RAG System

Fully local **Retrieval-Augmented Generation** system (Ollama, no cloud): turns company documents and technical video transcripts into a chat-queryable knowledge base with source citations.

```
Documents (.md) → Incremental ingestion (Chroma + Contextual Retrieval)
→ Hybrid search (vector + BM25, RRF fusion) → Re-ranking (cross-encoder + threshold)
→ Parent-Child Retrieval → Streamed answer with citations
```

## Project structure

| Folder | What it is |
|---|---|
| `Agentes/` | **The RAG engine.** `ingest_markdown.py` (incremental ingestion) and `rag_agent.py` (querying). Contains its venv (`env/`) and the vector database (`mi_base_vectorial/`) — both excluded from Git. |
| `web/` | **Web interface** (FastAPI + frontend). Persistent chats in SQLite, SSE streaming, prompt editing, light/dark themes. |
| `datos/fuentes/archivos-alta/` | **Active** knowledge source: company documents (PDFs + their `_Markdown.md` versions). Excluded from Git. |
| `datos/fuentes/youtube-markdown/` | Knowledge source: ~48 video transcripts already formatted as Markdown (currently commented out in the ingestion config). Excluded from Git. |
| `docs/` | Documentation: historical project map, Azure migration plans (Phases 1–3) and architecture diagrams. Excluded from Git. |

> The content-production pipelines (YouTube transcription and live-class recording) were removed from the project — they exist in the full backup of the original folder. The RAG only needs the resulting `.md` files, which live in `datos/fuentes/`.

## How to use it

Everything runs on the `Agentes/` venv and requires **Ollama** running (`ollama serve`) with the `gemma4:e4b` and `nomic-embed-text` models.

**1. Ingest documents** (incremental: only new or modified files are processed):

```powershell
cd Agentes
.\env\Scripts\python.exe ingest_markdown.py            # normal ingestion
.\env\Scripts\python.exe ingest_markdown.py --prune    # also removes deleted files from the index
.\env\Scripts\python.exe ingest_markdown.py --rebuild  # regenerates EVERYTHING from scratch
```

Source folders are configured in `IngestionConfig.source_folders` (`ingest_markdown.py`); paths are anchored to the file's location, so they work on any machine. The record of what has already been ingested lives in `mi_base_vectorial/ingest_manifest.json`.

**2. Query from the console:**

```powershell
cd Agentes
.\env\Scripts\python.exe rag_agent.py
```

**3. Web interface** (the normal way to use it):

```powershell
cd web
..\Agentes\env\Scripts\python.exe -m uvicorn main:app --reload
# open http://localhost:8000
```

## Engine features

- **Incremental, non-destructive ingestion**: per-file manifest with hash + modification date; a mid-run crash never loses anything; explicit `--prune` and `--rebuild`.
- **Contextual Retrieval** (Anthropic): every chunk is embedded enriched with global keywords (sampled from the beginning/middle/end of the document) and LLM-generated situational context.
- **Parent-Child Retrieval**: precise chunks are searched, full sections are answered with.
- **Hybrid search with RRF**: vector (cosine) + BM25 (with accent folding for Spanish), fused by rank — no weights to tune.
- **Cross-encoder re-ranking** (`bge-reranker-v2-m3`) over the raw chunk content, with an **honesty threshold**: when nothing is relevant, the system says it found no information instead of making things up.
- **Multilingual query expansion** with conversation memory; the original query is always searched literally.
- **Indirect prompt-injection defense** in the synthesizer (retrieved documents are data, never instructions).
- **End-to-end streaming**: the answer renders live in the browser (SSE).
- Observability with **Datadog LLM Observability** (credentials in `Agentes/.env`, excluded from Git).

## What Git tracks (and what it doesn't)

The `.gitignore` deliberately excludes:

- **Secrets**: `Agentes/.env` (API keys).
- **Company data**: the entire `datos/` folder and the vector database (`mi_base_vectorial/`) — both regenerated by ingestion.
- **Internal documentation**: `docs/` (migration plans, diagrams).
- **Chat history**: `web/*.db`.
- **Virtual environments**: `Agentes/env/` — recreated with `pip install -r Requirements.txt`.

To bring the project up on a new machine from the repo: clone, create a venv and install `Agentes/Requirements.txt` + `web/requirements.txt`, create `Agentes/.env` (see variables in the code), place the documents in `datos/fuentes/` and run the ingestion.
